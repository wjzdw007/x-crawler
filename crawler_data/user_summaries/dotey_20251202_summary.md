**新工具/技术**（如有）
- Nano Banana Pro：面向图像生成/编辑的强力模型，dotey 用它做手绘、3D 等风格化插画模板测试与分享。  
- Claude Code / subagent（及 CLAUDE md 实践）：子 agent 以 md 文件形式复用、隔离上下文，适合把复杂指令做成“技能”目录并在主 md 加索引。  
- Gemini（对比点）：在长输入/长输出的稳定性上被提为相对表现更好的一款大模型。  
- Anthropic 收购 Bun（新闻关联）：生态与工具链层面的组合，可能影响后端 JS/TS 开发体验。

**核心观点/方法论**（如有）
- 保持提示词/CLAUDE md 简短（建议不超过60行），复杂指令拆到独立目录并做索引，利于可维护性与复用。——重要性：降低提示复杂度、便于团队协作与版本管理。  
- 上下文可以很长但输出要短：一次不要让模型输出过多内容（建议几千字以内），否则幻觉上升，要用分页/分步输出。——重要性：权衡信息输入量与模型生成可靠性。  
- 采用“上下文工程 + 任务拆解”而非一蹴而就：把复杂任务分成多个子步骤/子 agent，先进行批判性评价或信息提取，再逐步输出成果。——重要性：提升质量、减少遗漏与错误。  
- 提示词迭代是创作劳动：从粗糙到通用模板需要多次试验，积累的工程化提示值得被认领与复用。——重要性：明确 prompt engineering 的价值，鼓励系统化沉淀。

**实践经验/案例**（如有）
- 分享了多套可直接复用的图像 prompt（如 3D chibi 店铺、赛博界面、等距居家办公场景），社区多次基于这些 prompt 产出并复刻。  
- 在图像生成上，原本为 GPT-4o 写的 prompt 在 Nano Banana Pro 上需要反复微调，说明不同模型之间需做适配与关键词调优。  
- 子 agent（subagent）实操：把角色、工具、模型与系统提示写成 md，放用户级或项目级目录，实现上下文隔离与可复用“武器库”。  
- 用 AI 做创意实验（如把店铺串成商业街、生成特斯拉/SpaceX 造型店）展示了 prompt 小改动能驱动大量有趣衍生作品，便于快速验证与传播。
