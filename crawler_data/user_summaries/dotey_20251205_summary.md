**新工具/技术**
- Nano Banana Pro：一个把提示词模板化、并能把抽象创意与实时数据可视化结合的生成式工具/工作流（擅长 3D/isometric 与奇幻视觉）。  
- Gemini（与 Nano Banana Pro 集成）：让模型能直接获取实时日期/天气等外部数据，消除了对外部 API 的部分依赖。  
- Claude Code / Codex：作为代码驱动的微调与自动化工具链，适合快速迭代与非程序员参与的场景。  
- Cursor / VS Code：被视作开发的辅助 IDE，顶级开发者把它们当成辅助而非主控工具。  
- Hugging Face hf-skills：新推出的“炼丹”插件，支持 SFT、DPO、GRPO 三种训练方法，可自动调度 HF 硬件并支持验证后导出为 GGUF 本地部署。  
- GGUF 本地部署：作为最终落地方案，方便把云端微调成果转为本地可用模型格式。

**核心观点/方法论**
- 设计可复用的模板，而不是写死的提示词：模板能适配多场景，降低重复劳动并提高用户可玩性。重要性在于把单次技巧变成可扩展的产品化能力。  
- 利用模型的原生能力（检索、世界知识、推理）：把能自动完成的部分交给模型，剩下的是结构与校验，能显著提升效率与体验。  
- 明确模型能力边界并在其范围内设计交互：最难的不是创意而是找到“可实现”的创意和模型能力的平衡点。  
- 把作品做成“框架/模板”而非“秀场作品”：让用户能替换输入并产生个性化结果，才能形成生态与传播效应。  
- Coding Agent 即管理角色比喻：使用 Coding Agent 更像做工程经理—靠任务拆分、能力边界与验收，而非微管理。

**实践经验/案例**
- 城市天气 GPTs 与 isometric 微缩场景：通过让模型拉取实时天气并生成 45° 俯视的 3D 风格城市画面，示范了把实时数据与视觉模板结合的工作流。  
- Nano Banana Pro 的模板被广泛复用：用户把模板改造成学校微缩景观、公司股票微缩场景等，说明模板化易于扩展。  
- Tarot / Javanese Primbon 等多文化提示词：展示了如何把同一结构适配不同文化/语言与主题（塔罗、农历/打古占卜等）。  
- 多模型并行策略：作者会并行让 GPT-5.1、GPT-4.5、Gemini 3 Pro 等模型尝试同一任务，再择优输出，提高成功率。  
- 微调与自动化落地链路：hf-skills + Claude Code/Codex 的组合可把复杂微调流程（验证、选硬件、微调方法）流水线化并导出 GGUF，降低部署门槛。
