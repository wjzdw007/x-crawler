**新工具/技术**
- Nano Banana Pro — 一套面向高质量图像生成的提示词模板集（多为 8K、电影级、微缩/裸眼3D 场景），便于快速复用和变体创作。  
- Veo 3.1 — 在多人协作/示例中作为配套模型/渲染栈被引用，用于提升视觉效果表现（collab 中出现）。  
- 高精度转录提示词（针对 Gemini 的规范化 Prompt） — 一份严格的转录规范，产出带发言人、时间戳和章节的可机读文本，强调一致性和可验证性。

**核心观点/方法论**
- 模板 + 实例 的提示工程范式 — 先定义元提示词（占位符、全局设置、负面提示等），再用具体实例替换生成多变体；提高可复现性与规模化创作能力。  
- 把提示词当作工程规范来写 — 明确相机视角、分辨率、渲染器、材质、文字渲染要求和负面提示，目标是把不确定性降到最低。  
- 强调文本/标签的鲁棒性与矢量化渲染 — 对儿童识图等场景特别要求矢量文字、字体和抗锯齿以避免乱码或像素化，体现“提示即规范”的思路。  
- 文化/场景适配策略 — 鼓励把主体与城市/季节/代表性饮品等本地元素结合，提升作品代入感与地域识别度。

**实践经验/案例**
- Cappuccino 微缩城市（示例：Chicago） — 用 45° 俯视的微缩城市场景漂浮在咖啡奶泡上，已被多人复现并延展为不同城市/饮品组合。  
- 裸眼 3D 大屏（L 型 LED） — 提示词能生成“突破屏幕边界”的立体动画效果，强调光照与在地投影以增强真实感。  
- 四季流转全景（上海外滩、悉尼歌剧院） — 用一张图无缝展示冬→春→夏→秋的过渡，体现光影与植被渐变的 prompt 写法。  
- 严格转录规范落地（视频转文本） — 提供可直接使用的转录提示，包含 ToC、章节划分、发言人一致性和时间戳格式，便于自动化流水线集成。  
- 社区验证与扩散 — 多位推友基于他的模板做了改动（如替换主体饮品、本地化标签），说明模板+实例方法在社区有良好可操作性。  
- 模型局限与应对措施 — 在文本/标签渲染上不同模型稳定性不一（如有人报告 Gemini 标签错标或乱码），对策包括更严格的负面提示、矢量文字与移除易误导的箭头等实践。
