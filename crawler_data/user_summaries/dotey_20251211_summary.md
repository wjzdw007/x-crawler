**核心观点/方法论**
- 提示词写作没那么复杂：把想要的视觉/功能细化成明确的约束（风格、构图、分辨率、元素），比抽象描述更可靠；这是提高产出的关键。
- 针对小众模型要现实预期：主流大语言模型目前对 nano banana pro 类专属提示词训练数据不足，不能完全替代人工调优；要么用相近模型生成初稿再调整。
- 明确输出规格很重要：在 prompt 中写清分辨率（如 2K）、纵横比、光照、细节等级、不要的元素（--no）等，能显著减少废图和后期改动成本。
- 高质量工作流要成文：开启 Thinking / Web Grounding、选好构图比例、用图片辅助（将文本限制为关键信息）、给画面留白并考虑视觉层次，能稳定产出高质作品。
- 分享成品与提示词形成闭环：把完整 prompt + 生成结果公开，能快速积累可复用模板并带来传播效果（流量与同行反馈）。

**实践经验/案例**
- 成果证明：dotey 通过数百张 Nano Banana Pro 图片和提示词积累了几百万浏览，说明“示例+模版化提示词”对个人影响力很有效。
- 可直接复用/改写的模板：他/转发的示例包含“Then vs Now（时代对比）”“人鱼咖啡馆”“漂流瓶里的黑珍珠”“企业垂直演进”等，都是可按场景替换元素的高复用 prompt。
- 实操技巧：当目标模型生成能力不足时，可先生成 DALL·E / MJ / SD 风格的提示词或草图，再人工调整以适配 Nano Banana Pro，从而缩短迭代周期。
- 工具联动实例：GPT-5.2 已在 Codex CLI 上可用，表明新模型先在开发者工具链中落地，适合把生成/调试集成进创作流水线。
- 社区运营经验：发布博客、作品合集和 prompt 线程（并鼓励交作业/转发）能放大传播、获取实测反馈并形成可持续的提示词库。
