**新工具/技术**（如有）
- NotebookLM（Flash Cards）+ 能从你上传的素材自动生成问答式记忆卡片，默认只显示问题，点开才显答案，适合复习与教学记忆。  
- nano banana pro（绘图/提示词套件）+ dotey 提供的多种画图提示模板（扁平涂鸦、信息图、植物拼贴等），方便快速产出一致风格的插画。  
- Claude Code / Amp / Pi / Codex / GitHub Copilot + 多种 AI 编程工具的对照：不同定位（精致/实惠/黑客玩具/主力/辅助），实际使用存在明显“vibe”差异。  
- Gem/Opal（工作流生成器）+ 可把多步流程自动化，但当前与 Gemini 3 的兼容性受限；实操中用 JSON 打包内容能加速后续页面生成。

**核心观点/方法论**（如有）
- Vibe Coding：选择工具更多靠“感觉/vibe”而非单一性能指标，重要在于工具与个人工作流的契合度，会影响效率与满意度。  
- 人机关系与记忆：当前模型有“记忆”与人格化倾向，需谨慎处理情感投射与责任归属，这决定了协作边界与信任建立。  
- 可追溯性与提示词版本控制：生成式代码需要把 prompt / 对话历史纳入版本控制，否则无法复现或评价改动质量。  
- 新的审查与可观测性需求：AI 生成代码的评审流程、监控与调试工具需重新设计以适配人机协作的特殊性。

**实践经验/案例**（如有）
- NotebookLM 的 Flash Cards 实操证据：可直接把素材转成问题/答案卡片，适合中学生和复习场景，用户反馈好用。  
- PPT 自动化（社区实操）：有人用 Gem/Opal + JSON 打包的办法，把 dotey 的多步提示词流程自动化，减少逐页粘贴的重复工作。  
- 插画提示词可直接用“uploaded photo”作为 Scene 控制参考，配合 nano banana pro 模板可稳定输出扁平涂鸦/信息图风格。  
- Vibe Coding 教学要点：要教好 Vibe Coding，前提仍是扎实的 coding/软件工程能力——提示词只是表象，关键在任务拆解、验证与调试能力。
