**新工具/技术**（如有）
- Prompt 模板化（JSON / template + input_variables）：把长提示词做成可复用的模板，变量集中管理，便于批量生成和版本控制。  
- XML / 自定义标签（如 <context>、<think>）：用标签把提示词分块，既清晰给人看，也能帮助模型把不同信息隔离开来。

**核心观点/方法论**（如有）
- 不用纠结括号的“严格语义”：中括号、花括号、${} 在模型里主要起占位/变量提示作用，模型能根据上下文理解它们的意图。重要性：降低写提示的心智负担，提高迭代速度。  
- 用小括号做补充说明：把对词语的背景或注释放在（）内，便于在生成时让模型输出带解释的内容。  
- 把提示词当代码来写：结构化（标签/模板/变量）能减少歧义、提高可复用性和可维护性，对复杂/超长提示尤其重要。

**实践经验/案例**（如有）
- 烟花提示词模板（dotey 提供的例子）：用 {Subject} 占位符生成“由烟花拼出特定字符/图形”的超写实场景，要求高分辨率、物理合理的光效与无额外文字，便于直接喂给图像生成器。  
- 变量放底部 / JSON 化管理：把风格、主体等变量列在模板底部或 input_variables 中，替换即可复用同一模板生成不同结果，减少手工修改错误。  
- XML 标签的实际好处：把背景/context、参考图、系统指令等用标签包起来，像收纳盒一样防止信息互相污染，既利于人审阅也利于模型精确渲染（DeepSeek 用 <think> 举例）。  
- 使用时的注意：除非协议/工具要求严格格式，否则不必每次手动删括号；但要注意不要把“空内容”错误渲染进 prompt（会污染上下文，影响稳定性）。  
- 推荐流程：把角色与任务写清在 system prompt，用模板+变量输入做前端，用结构化标签分割背景与生成指令，从而兼顾可读性、可复用性与生成稳定性。
