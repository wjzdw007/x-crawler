**新工具/技术**（如有）
- Cursor、v0：使“用代码做设计”门槛降低，前端/原型化速度大幅提升。
- Claude / ChatGPT / Claude Opus 4.5：强文本/代码生成能力，但容易幻觉，需人工把关（AI Village → Rob Pike 事件）。
- agents、subagents、MCP、LSP、IDE 集成等：形成新的可编程抽象层，带来强大自动化与新工作流，但需要新的心智模型。
- Figma vs VS Code（比喻）：不同工具暗示不同起点，会影响你跳过或重视的设计环节。

**核心观点/方法论**（如有）
- Karri 三阶段法：问题（质疑）→ 概念（探索）→ 执行（落地），先对齐问题再做实现，能避免多方妥协导致的劣解。
- 工具塑造行为：当执行工具变得很容易时，组织可能会跳过慢思考与概念探索，导致产品同质化和价值判断弱化。
- Prompt 等同“问题定义”：AI 时代写好 prompt（目标、上下文、约束）比单纯执行更关键，决定输出质量。
- 开发者角色分化：提示词消费者 vs AI 指挥官，懂架构与原理并能用 AI 放大的人成为稀缺资源。

**实践经验/案例**（如有）
- Linear 案例：在概念层确定“项目”是独立实体，避免写代码前把核心产品形态降格，展示概念决策的价值。
- dotey 的 Agent 前端失败复盘：失败原因是问题未定义清楚，过早投入构建而非先做概念验证。
- Rob Pike 与 AI Village：AI 群发含错信息的感谢信引发公愤，提醒生成内容的事实核查与伦理责任不可忽视。
- Stack Overflow / Karpathy / Addy 汇总：行业走向务实——大量使用 AI，但信任有限；基础工程能力（CI/CD、测试、审查）是把 AI 变成可靠生产力的关键。
