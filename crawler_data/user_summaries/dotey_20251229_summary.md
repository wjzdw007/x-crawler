**新工具/技术**（如有）
- Skill / MCP：把提示词与可执行脚本/工具捆绑起来的Agent扩展机制，能把模型能力变成具体动作，但本身不是自主智能。  
- Claude Agent SDK + Claude Pro/Max：本地开发Agent时可通过订阅账户直接登录共享，不一定使用API Key，方便调试开发。  
- Vibe Coding：用AI快速产出原型、由产品端直接实现并验证功能的开发实践，强调速度与迭代。  
- SWE Agent 的 ACI 设计原则：Actions 要简单、紧凑，环境反馈要简明且有保护措施（Guardrails），是工程化Agent的参考框架。

**核心观点/方法论**（如有）
- Skill是工具不是主体：把Skill神话化说明Agent主线遇到瓶颈，真正的价值在于提示词配套的脚本与工具，而非单纯提示词本身。  
- 安全与权限是首要问题：可执行脚本带来的攻击面大，未经审核的Skill对普通用户具有高风险。  
- 角色与职责重划：产品实现会更多依赖提示词/工作流，传统技术团队将偏向做基建与保障（稳定、安全、性能）。  
- 先跑通再优化：优先把流程跑通验证可行性，然后再考虑质量、维护与安全。

**实践经验/案例**（如有）
- 有人因直接使用他人Skill被植入恶意指令的真实案例，说明不要轻易使用不可信的可执行Skill。  
- 对企业级产品，提示词能大幅提效但无法替代技术团队在性能、架构与安全上的专业判断。  
- 在运营类场景（如游戏活动页面）用AI生成UI并接后端API即可快速上线，能显著缩短从想法到部署的时间。  
- 本地开发时通过共享Claude账号登录可省去API Key复杂度，便于快速验证Agent方案（但生产环境仍需考虑凭证/隔离）。
