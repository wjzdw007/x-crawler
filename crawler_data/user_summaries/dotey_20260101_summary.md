**新工具/技术**
- notebooklm（fast research / deep research）— 用于自动收集、整理专题资料，能把主题研究中 ~80% 的信息自动抓取并初步汇总，剩余 20% 做人工精挑细选。  
- 推理模型（如 OpenAI o1、DeepSeek R1）— 借由“慢思考”训练，让模型内部分步推理、验证与修正，显著提升 Agent 在编程与数学类任务的可靠性。  
- 编程 Agent / CLI 工具（Claude Code、Codex CLI、各厂商 CLI）— 把编程能力包装成异步云端或本地 CLI，能接收任务、自动运行并返回 PR，已成为高产值产品形态。  
- AI 基础设施发电方案（燃气轮机、往复式发动机、燃料电池、Megapack 电池等）— 数据中心“自带发电”成为短期加速上线的实际手段，设备类型多样对应不同交付/成本/审批权衡。

**核心观点/方法论**
- 成功不只是“做对事”，而是“没做错事”——像 Manus 那样的反直觉决策（不训模型、先服务付费用户、出海）背后是经验、资源与风险承受能力的组合。  
- 速度是护城河（Speed is the moat）——在算力竞赛中，提前上线几个月的时间价值能远超单纯节省成本。  
- 电力已成为 AI 的主要瓶颈——不是芯片或数据，而是能否及时、稳定地获得大规模电力供应，影响架构与商业节奏。  
- Agent 实用化的关键在于可验证闭环——当模型能自我分步推理并运行、检验代码/结果时，Agent 才能实际替人“干活”。  
- 开源与地缘竞争重塑格局——中国开源模型崛起与多地工具互补，改变了原有美国主导的生态与投资预期。  
- 安全风险正在被“常态化”——宽松地让 Agent 自主执行（YOLO 模式）短期看易用，但长期有引爆式风险，需建立更严格的合规与测试机制（conformance suite）。

**实践经验/案例**
- Manus 案例：靠连续创业者的经验、VC 背书和早期盈利做出激进选择，说明资源/底气决定可行的策略组合。  
- xAI / 马斯克：用租赁燃气轮机和大量 Megapack 电池“自建发电”快速上线算力，证明时间价值可以抵消更高的能耗与运维成本。  
- Anthropic 的 Claude Code 与 CLI 变现：一个命令行工具即可创造大规模营收，说明把模型能执行的能力封装为开发者工具商业价值极高。  
- notebooklm 实操：用 fast/deep research 把大量初步资料自动化抓取，能把研究中繁琐的检索与整理环节大幅压缩，剩余由个人笔记/专库补齐。
