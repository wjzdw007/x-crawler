**新工具/技术**  
- Claude Agent SDK — 适合单智能体开发，开箱即用但通常与 Claude 生态绑定。  
- LangGraph / CrewAI / Agno — 常被推荐用于多智能体编排和协同的框架。  
- DeepSeek R1 — 一款开源前沿模型，证明开源也能逼近顶尖能力，便于本地化与定制。  
- NVIDIA Blueprint (DGX Spark 演示) — 构建 Agent 的参考框架，支持本地/云模型混合与意图路由。  
- Claude Code / Claude Desktop — Claude Code 提供 coding agent 能力，Desktop 集成带来图形化交互体验。  
- OpenCode — 面向团队的开源、模型无关的 coding agent，便于定制化工作流。  
- Cursor 编程工具 — 英伟达内部示例，展示 Agent 能承担从规划到写测、修复的编码闭环。  
- Perplexity 的多模型调用实践 — 说明在推理链不同环节用不同模型能提升结果质量与效率。

**核心观点/方法论**  
- 先把单智能体做好再考虑多智能体：多智能体复杂度高且当前缺少成熟最佳实践。重要因为能节省大量弯路与工程成本。  
- Agentic AI 从“会聊天”变成“能干活”：它要会推理、用工具、规划并协作，才能替人完成真实任务。  
- 推理模型（test‑time scaling）是可控、可靠 Agent 的基础：更多推理时间换更高回答质量与更少幻觉。  
- 开源模型是民主化力量：降低门槛，让公司/团队可基于开源构建定制 Agent。  
- 企业采用的主要阻力是容错率与合规：需要从容错小的场景切入，逐步扩大应用范围。

**实践经验/案例**  
- 找需求靠观察流量聚集地（Reddit/Twitter/评论区/小红书），把重复抱怨当做强信号，先做 MVP 或付费快闪课程验证。  
- NVIDIA 的 DGX Spark 演示是实战范例：用本地模型处理隐私敏感邮件、用 frontier 模型做复杂推理，意图路由自动分配模型，落地路径清晰。  
- 编码工具策略：个人用户用官方 CLI（Claude Code）即可快速迭代；团队则更偏好开源/模型无关的方案（如 OpenCode）以固定并可扩展协作流。
