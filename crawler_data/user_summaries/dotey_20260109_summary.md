**新工具/技术**
- Claude Code：Anthropic 的编程/Agent 平台，支持 skills、plugins 和 subagent，目标从编程向“通用 Agent”扩展，可组合性强。  
- Codex CLI：以模型接管为主、可定制性较少，擅长纯编程任务但不利于构建可移植工作流。  
- code-simplifier agent（Claude Code 插件）：开源的代码简化 Agent，适合在长会话或复杂 PR 后做清理与简化。  
- Skills（生成图片的 Skill）：通过脚本 + 第三方 API（如 nano banana pro）实现图像生成，可被其它 skills 调用，实现模块化图像能力。  
- Bash 工具（Claude Agent SDK）：把中间结果落地为文件的工具链，提升可复现、可验证和可组合性，并配套权限控制。  
- Dify（可视化 workflow 编排）：以 DAG/拖拽为主，确定性与稳定性强，适合高频、企业级 SOP 场景。  
- MCP/skills/plugins 协议思路：Anthropic 借标准协议而非开源构建生态，强调可接入的本地/服务端能力互通。

**核心观点/方法论**
- 模型“接管一切”与可定制化之间的取舍：完全自动化能提效但难以满足复杂定制需求，工程上常需把控制权和可说明性留给人/脚本。  
- Agent + Skills 的组合优于单纯流程编排：把流程转成可复用的 skill 可以在灵活性与分享、移植性之间取得更好平衡。  
- 用自然语言编排工作流：把复杂能力拆成单一 skill/subagent，并用自然语言描述上层工作流，降低编码门槛同时保留可定制性。  
- 本地文件系统与文件路径传参：把中间结果存为文件、subagent 仅传文件路径，能显著降低上下文开销并提高可验证性。  
- 可复现/可验证优先于模型“脑算”：落地中间产物（文件、日志）便于调试、审计和回溯，尤其在财务、合规类任务重要。  
- 设计 Skill 仍需要编程和软件工程能力：skill 不只是 prompt，还包括脚本、工具接入与文档，非纯自然语言即插即用。  
- 安全与权限必须并行：可执行命令（如 Bash）带来强能力同时带来风险，需分级权限、解析器与用户确认机制。  
- 长时运行与重构靠传统工程支撑：重构屎山代码或做长期任务，自动化测试与模块化迁移不可或缺。

**实践经验/案例**
- 写作与配图：dotey 用 Claude Code + Skill 做写作辅助并自动生成配图，证明 Agent 可跨编程场景落地。  
- 代码清理：用 code-simplifier 在会话末或复杂 PR 上自动简化/重整代码，提高审查效率。  
- 自动发文 Skill：已有 X（Twitter）自动发布 Skill，可以扩展为公众号等平台的发布工具（脚本控制浏览器 + 剪贴板）。  
- 迭代与自我进化：通过本地文件存储、subagent 与 ralph-loop 可以实现 subagent/system-prompt 的反复迭代与优化（需注意成本）。  
- 长时任务示例：代码迁移、逆向、带完整测试集的模块重构等仍适合用 Codex/长跑型 Agent + 自动化测试来执行。  
- Dify 与 Skills 的互补：对于稳定、高复用的 80+ 节点流程，Dify 更稳；也可以把复杂节点封装成脚本，再由 skill 调用以兼顾稳定与灵活。
