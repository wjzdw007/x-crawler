**新工具/技术**（如有）
- Nano banana Pro：针对图像生成的模型/样式器，作者认为在生成小红书风格图片时效果最好，可结合 Gemini Pro 使用。
- GEM（作者分享的 GEM）：一套可直接复用的 prompt/配置包，方便在大模型里批量生成图文提示词与素材。
- Gemini Pro / Gemini：用于生成图片的模型，支持批量生图（可把所有生图提示词做成大 JSON 批量喂入）。
- Claude Cowork（Anthropic）：把 Claude Code 能力做成图形化桌面助手，能读写本地文件夹、调用 Skills，并接入 Drive/Slack 等连接器。
- VSCode/Skill 行读取约定：社区经验指出 AI 读取 Skill 时按行处理，VSCode 默认读取前 100 行，影响 skill 的编写方式。

**核心观点/方法论**（如有）
- Agent + Skills 更容易进化：Agent 在会话中既是执行者又是“本地开发者”，能直接定位输入/输出和工具调用，缩短反馈闭环并能即时修复或优化 skill。
- 文本化的 Skills + git 版本控制是关键：把技能以文本文件管理便于记录变更、回溯问题并把个人改进共享给他人，形成可复制的进化路径。
- 模块化提示词与批量化生成：把内容拆成每页/每张图的独立提示词，用大模型生成提示词，再批量喂图，便于微调和规模化产出。
- 风险与兼容性并重：Cowork 等工具需要明确授权，注意提示词注入、误操作风险；虚拟机/架构差异会导致依赖本地二进制或 node_modules 的 skill 无法直接运行。
- 基础模型迭代极快：排行榜数据表明顶级模型优势窗口被压缩到以“月”为单位，产品设计要适应基础能力快速变化的现实。

**实践经验/案例**（如有）
- 小红书信息图流水线：用作者的提示词把输入拆成每页提示词，生成独立提示词后在 Gemini Pro + Nano banana Pro 批量产图，效果与可控性都较好。
- 会话内修复 skill：在当前会话让 agent 检查提示词或脚本并修复，实测能快速定位问题并改进输出结果，显著缩短修复时长。
- 批量生图与下载：用户实践把所有生图提示词转成一个大 JSON 喂给 Gemini，实现批量生图；配合浏览器插件可一键下载无水印图片。
- Skill 编写技巧：鉴于 AI 按行读取且 VSCode 默认关注前 100 行，建议在前 100 行内把关键说明写清或调整格式，避免被截断影响行为。
