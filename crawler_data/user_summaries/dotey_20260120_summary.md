**新工具/技术**
- Craft Agents — 基于 Claude Agent SDK + Electron 的开源图形化 Agent，目标把 Claude Code 的能力用更友好的 UI/UX 封装，便于非程序员协作与定制（Apache-2.0）。
- Skills（Agent Skill）— 把一套重复可复用的“最佳实践”固化为可调用的能力模块，便于自动化反复任务。

**核心观点/方法论**
- Skill 的核心价值在于复用：先手动跑通流程、沉淀最佳实践，再把步骤固化为 Skill，最后在实战中持续迭代优化——这样才能把一次性工作变成长期效率提升。
- 不要为了做 Skill 而做 Skill：如果是一次性任务或构建成本高于未来节省时间，就不值得构建。
- 让 AI 帮你写 Skill：AI 更擅长把提示词 + 脚本组合成可执行的 Skill，优先用 Agent 自动生成而不是手工编码。
- xAI 的路线启示：小模型 + 速度优先是一种可行策略，强调快速迭代和低延迟带来的产品可用性与商业价值。
- 部署思路重要性：把算力部署到边缘/闲置资源（如 Tesla 汽车）能构成战略优势，但同时极其敏感，暴露即带风险。

**实践经验/案例**
- 文档自动化场景（doc/docx → 生成新文档）：推荐流程——先手工多跑几次找出固定步骤/坑点→在支持 Skills 的 Agent 中完整演示一次→让 Agent 把操作生成 Skill→用 Skill 执行并持续改进。
- Craft Agents 案例说明：用图形界面把复杂命令行能力下沉到非工程团队（财务/市场/客服/HR），证明 UI 能显著扩大工具受众与日常采纳度。
- xAI（Sully 泄露）案例教训：公开核心产品概念（Macrohard 人类模拟器）、部署计划（租用 Tesla 闲置算力）、模型策略，会把竞争优势和商业机密直接暴露给对手，带来法律/战略/运营后果。
- 商业影响提示：战略级部署（如车联网算力）和内部测试细节被泄露后，可能需重新评估合作条款、隐私/合规与客户沟通策略。
- 简单决策准则（Noctrix 补充）：如果构建 Skill 的时间 > 未来一年内手动执行该任务节省的时间，就别建，除非你打算分享给别人用。
- 操作性建议：把 Skill 当作“可被数据驱动和迭代的资产”，每次用后检查输出并让 Agent 更新 Skill，以降低初期设计偏差的长期成本。
