**新工具/技术**（如有）
- jimliu/baoyu-skills（Slide Deck）：一次性生成多份提纲并导出 pptx，支持多风格与自定义，生成后保留原始提示词便于迭代。  
- Claude Code / Opus 4.5 Tasks：用 Tasks 取代旧的 Todo，支持任务依赖、多 Agent 与跨会话状态广播，任务可本地存储（~/.claude/tasks）。  
- Claude SubAgent Task 并行执行：让同一系统同时开多个后台子任务用于批量操作，配合试点再放量可加速批量更新。  
- 两段式台历/设计工作流（Visual Design Skill）：用脚本先生成结构化骨架（CSV），再把骨架变量塞入绘图模型以保证准确性与审美自由度。

**核心观点/方法论**（如有）
- 一个会话只做一件事：避免把“顺手问一下”塞满上下文，能降低信息混乱和上下文耗尽的风险，便于后续压缩与管理。  
- 用“by the way”做预热：小心构造的顺带问能平滑地把注意力从旧主题迁移到新主题，但这更像是技巧而非常规工作流。  
- Token＝流量套餐的类比：把 token 当作“流量”看待，降低成本和提高可用性（像包月）会是 AI 应用爆发的前提。  
- 关于加班与岗位选择（转发议题）：加班有时不可避免，理解自己在组织里的角色与边界，以及可以通过换岗/避开管理来降低长期压力很重要。

**实践经验/案例**（如有）
- Slide Deck 实操：先选风格/受众/数量与是否需要确认，系统可生成多个提纲供人工挑选和微调，减少来回迭代成本。  
- Tasks 在工程协作里的价值：任务依赖与跨会话广播能把长期、跨 agent 的工程流彻底打通，便于复杂项目管理与审计。  
- 并行子任务的使用方法：先对 1–2 个样本做试点，确认无误后再批量运行，最后保留人工审查以防自动改动引入错误。  
- 两段式生成避免“幻觉”与错位：用代码保证逻辑（日期、表格结构等）无误，再把美学交给模型，既稳又有创意。  
- 日常健康/抗压实践（转发）：在高强度工作环境下保持基础保健（多喝水、定时活动、骑行/步行）是降低突发健康风险的实用手段。
