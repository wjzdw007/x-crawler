**新工具/技术**（如有）
- nanobot：香港大学数据科学实验室开源的超轻量级个人 AI 助手（约4000行代码），支持多 LLM 提供商、本地模型、多渠道接入和 Skills 扩展，适合学习与快速部署。
- Claude Code 的 /insights 指令：自动读取过去一个月消息，生成项目复盘、使用分析和可执行的工作流优化建议（建议把部分流程做成 Skills）。
- AI短剧制作管线：以 AI 为创作引擎，把网络小说自动化生成短剧视频，常见商业化模式为首集免费、后续付费。
- VoxCat 的风格化修复 prompt 模板：把文化遗产修复的色彩/质感/发型/光影等要素列为“硬约束”以保证高保真重现。

**核心观点/方法论**（如有）
- AI驱动的内容生产同时是创作工具和商业创新点：把创作效率变现（如短剧付费模式）是可复制的创业路径。
- 强约束的 prompt 设计比随意描述更关键：对文化、质感、光影设定硬约束，能提高生成结果的可控性与可信度。
- 轻量化、模块化（Skills/Agents）优先：可读代码、可组合的能力便于迭代、二次开发和团队落地。
- 多模态与多渠道兼容性是产品化关键：支持多模型、多接口和多终端能把研究成果快速推向用户场景。

**实践经验/案例**（如有）
- nanobot 典型部署路径：pip install nanobot-ai + 配置 JSON 填入 API Key，可 CLI 聊天或挂到 Telegram/WhatsApp/飞书，支持 Docker 生产化部署。
- nanobot 适合教学与二次开发：代码量小、结构清晰，方便研究 Agent 架构与扩展 Skills。
- Claude Code /insights 的实测结论：能给出靠谱优化建议，实际建议将重复性工作抽象为 Skills 以提高效率。
- 网文作者转型案例：已有作者用 AI 把小说拍成短剧并采用“首集免费、后续付费”模式，说明 AI 可显著降低制作门槛并催生新商业化路径。
- VoxCat 的 Dunhuang 案例：通过细粒度风格设定实现文化遗产的视觉重建，示范了在敏感文化题材上使用严格 prompt 的可行性与风险控制方法。
- 小结（落点）：当下趋势是把轻量可读的 Agent 框架、明确的 prompt 规范和可货币化的内容流结合，优先做可组合的 Skills 与多渠道部署以快速验证商业模式。
